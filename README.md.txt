# Actividad: Ingeniería de Requisitos - Gestión de Proyectos de Software

## 📋 Introducción

Este repositorio contiene toda la documentación y entregables desarrollados para la **actividad de Ingeniería de Requisitos** de la asignatura **Gestión de Proyectos de Software**.

El objetivo principal fue aplicar las técnicas y buenas prácticas de elicitación, análisis, especificación, documentación y gestión de requisitos de un sistema software.

### 🎯 Objetivos de la Actividad

- Elaborar el Acta de Constitución del Proyecto (Project Charter).
- Identificar y documentar requisitos funcionales y no funcionales.
- Realizar el Catálogo de Requisitos y Product Backlog.
- Desarrollar la Especificación de Requisitos de Software (SRS).
- Modelar el sistema mediante Casos de Uso y User Stories.
- Crear glosario de términos y otros documentos de apoyo.

### 📁 Estructura del Repositorio

- `Acta_de_Constitución_del_Proyecto_(Project_Charter).docx`
- `Especificación_de_Requisitos_(SRS).docx`
- `Casos_de_Uso_y_User_Stories.docx`
- `Catálogo_de_Requisitos_y_Backlog.docx`
- `Especificación_de_Requisitos_No_Funcionales.docx`
- `Glosario_de_Términos.docx`
- Otros documentos de apoyo.

### 🛠 Metodología

Se aplicaron conceptos vistos en clase de Gestión de Proyectos de Software, siguiendo recomendaciones del estándar **IEEE 830-1998** para la especificación de requisitos.
