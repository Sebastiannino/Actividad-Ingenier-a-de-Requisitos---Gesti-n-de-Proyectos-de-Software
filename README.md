## Objetivo del Repositorio

Este repositorio centraliza la documentación generada a lo largo de las diferentes fases del ciclo de vida del desarrollo de software, incluyendo **Ingeniería de Requerimientos**, **Fase de Desarrollo**, **Fase de Pruebas** y **Fase de Implementación**. Los documentos aquí presentados son formatos estándar y ejemplos de cómo se aplican técnicas de elicitación, análisis, especificación, documentación, trazabilidad, pruebas y despliegue para el desarrollo de un aplicativo con **Inteligencia Artificial** para la optimización de procesos empresariales de la **Empresa Tecnológica Innovación S.A.S.**

---

## Metodología

Se utilizó la **metodología Scrum** con sprints de 2 semanas. Se siguieron buenas prácticas de Ingeniería de Requisitos y estándares como **IEEE 830**, **ISO/IEC 25010** (Calidad) e **ISO/IEC 27001** (Seguridad).
